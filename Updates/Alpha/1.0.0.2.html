Version 1.0.0.2 Alpha

Tuesday, May 13th 2025

Revision 2

Depreciated:

  • system.command()

Aesthetic:

  • Commands requiring specific parameters now return a brief warning describing the expected format.

Functional:

  • Added support for specialized project files referencing multiple source files.
  • Actions, objects, and variables can be declared with expressional parameters using the 'default?' keyword.
  • Collection based commands now return results more appropriately.
  • Directory, file, and path commands now return false if a command is unable to be completed.
  • 'math.random()' command for working with random numbers.
  • Task-based and asynchronous commands using the 'task' and 'await' keywords.
  • Try logic using default, catch, and continue groups, and custom arguments.
  • Variable property shortcuts using let, get, and set groups.

Bug Fixes:

  • Fixed an issue where source files were being improperly imported.
  • Reworked switch and case statement logic.

Environment:

  • Converted storage systems to use a variable class that wraps values with properties and methods for more advanced data retrieval.
  • Increased documentation.
  • Migrated methods to utility classes for increased maintainability and future development.
  • Merged several text utilities into collections methods with similar use-case.
  • Reworked comments and line execution system for future updates.
  • Tagging system for more efficient definition recognition.
  • Wrapped logic for several commands with exception handling for easier development.
