Version 1.0.0.1 Alpha

Wednesday, April 30th 2025

Revision 2

Aesthetic:

  • Tweaked action execution, variable definition, and implicit formatting to improve code write-ability.
  • Collections can now be defined using "[]" initializers at the end of their defined name. For example: "var list[]".
  • Console and debug command now implicitly use the subcommand "line" by default.
  • Added "!" and "?" logic for binary results and void checks.
  • Expanded upon and refined statement expression terms.
  • Implemented warning messages and a warning log.
  • Improved warning message formatting.
  • Added line positions to build messages.

Functional:

  • Implemented restricted syntax for definitions.
  • Added default value returns for binary values and voids.
  • Added logic for collections.
  • Added support for collection command and formatting methods.
  • Added action, object, and spacing instancing.
    ** Please note this feature is currently untested.
  • Added "case" and "default" statements for switch logic.
  • Reworked comparative logic for if statements.
  • Completed implementation of date/time commands and formatting.
    ** This feature is only partially tested.

Bug Fixes:

  • Fixed file-system management logic when appending and writing.

Environment:

  • Modularized block definitions, and reworked Structure.cs for block instancing.
  • Added Void() class to allow for variable that are defined by name only.
